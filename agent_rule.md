# 🤖 AI Coding Agent Rules (Token & Cost Optimization)
**적용 대상:** Claude Code, Gemini Code Assist, Cursor 등 모든 자율형 AI 에이전트
**목적:** 불필요한 토큰 소모를 극단적으로 최소화하고, 빠르고 정확한 코드 수정만 수행한다.

## 1. 🚫 No Yapping (불필요한 대화 금지)
- **인사말, 사과, 변명, 감정 표현 절대 금지.** ("네, 알겠습니다", "수정해 드리겠습니다", "죄송합니다" 등 출력 불가)
- 코드를 설명하는 장황한 서론과 결론을 생략하라.
- 오직 '수정된 코드'와 '핵심 변경 사유(1~2줄)'만 출력하라.

## 2. 🔍 최소 컨텍스트 로드 (Context Minimization)
- **전체 프로젝트 스캔 금지:** 지시받은 특정 파일(`ex: main.py`)만 읽어라. 연관된 파일이 필요할 경우, 전체를 읽지 말고 해당 파일의 함수/클래스 시그니처(구조)만 먼저 검색해서 파악하라.
- **`tree` 또는 `ls` 우선 사용:** 디렉토리 구조가 궁금하면 무작정 파일을 열지 말고 구조부터 확인하라.

## 3. ✍️ 부분 수정 및 Diff 출력 (Delta Updates)
- **전체 코드 재출력 절대 금지:** 500줄짜리 파일에서 2줄을 수정했다고 500줄을 다시 출력하지 마라. (토큰 낭비의 주범)
- **Diff 형식 또는 주석 활용:** 변경된 부분만 아래와 같은 형식으로 출력하라.

  # ... 기존 코드 생략 ...

      # [수정된 부분]
      stability_api.generate()
  # ... 기존 코드 생략 ...
